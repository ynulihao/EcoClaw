## 🌐 概述

**EcoClaw** 是一款面向 [OpenClaw](https://github.com/nicepkg/openclaw) 的省钱插件。它基于 [PinchBench](https://github.com/pinchbench/leaderboard) 真实评测数据，自动为每条 prompt 匹配便宜又好用的模型。

### 🌟 核心特性

- **最高省 98% 的 LLM 费用** — 基于真实评测数据，自动为每个任务匹配便宜又好用的模型，覆盖 23 个任务类别和 3 种省钱策略。
- **零配置，数据常新** — 开箱即用；评测数据每 6 小时自动刷新。
- **自动降级容错** — 主模型返回 429/5xx 时，自动切换到备选模型重试。

## ⚙️ 工作原理

```
用户发送 Prompt
    │
    ▼
┌─────────────────────────────────────────────┐
│  ① 分类                                     │
│  prompt → embedding → 余弦相似度             │
│  与 23 个预计算任务向量比较                    │
│  → 任务类别                                  │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  ② 打分                                     │
│  任务类别 + 省钱策略                          │
│  → 加权综合分 (质量 × w + 成本 × w)          │
│  → 对所有模型排序                             │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  ③ 选择                                     │
│  最优模型 + 2 个备选 → OpenRouter 转发        │
│  → 流式返回响应给用户                         │
│  (429/5xx 自动重试)                          │
└─────────────────────────────────────────────┘
```

## 🎯 省钱策略

| 模型 ID | 策略 | 质量权重 | 成本权重 | 说明 |
|---------|------|---------|---------|------|

| `ecoclaw/balance` | 均衡（默认） | 50% | 50% | 最佳平衡点：最多省 96%，质量接近最优 |
| `ecoclaw/eco` | 成本优先 | 20% | 80% | 极致省钱 — 最多省 98% |


## 🚀 快速开始

### 前置要求

- **Node.js** >= 20（`httpProxy` / `httpsProxy` 出站代理依赖 Node.js 24.5+）
- **OpenRouter API Key** — 用于 embedding 计算和模型调用
- **OpenClaw** >= 2026.3

### 安装与运行

```bash
# 1. 克隆 & 安装依赖
git clone https://github.com/ynulihao/AgentSkillOS.git && cd EcoClaw
 && cd EcoClaw
npm install

# 2. 配置 OpenRouter API Key / 出站代理
#    方式 A（推荐）：在项目根目录创建 secrets.local.json
cat > secrets.local.json <<'EOF'
{
  "openrouterApiKey": "sk-or-v1-你的key",
  "httpProxy": "http://127.0.0.1:7890",
  "httpsProxy": "http://127.0.0.1:7890",
  "noProxy": "127.0.0.1,localhost"
}
EOF
#    方式 B：通过环境变量
export OPENROUTER_API_KEY=sk-or-v1-...
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
export NO_PROXY=127.0.0.1,localhost

# 3. 拉取模型快照（验证 OpenRouter 可用性）
npm run pull:snapshot

# 4. 构建插件（自动先生成 embedding 向量）
npm run build

# 5. 以开发模式安装到 OpenClaw（symlink，方便迭代）
openclaw plugins install -l .

# 6. 重启 gateway，选择模型
openclaw gateway restart
openclaw models set ecoclaw/balance
```

> `secrets.local.json` 已被 `.gitignore` 排除，不会提交到仓库。

## 🔧 配置

### API Key 与出站代理

优先级：`secrets.local.json` > 环境变量 `OPENROUTER_API_KEY`、`HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY`

```json
{
  "openrouterApiKey": "sk-or-v1-你的key",
  "httpProxy": "http://127.0.0.1:7890",
  "httpsProxy": "http://127.0.0.1:7890",
  "noProxy": "127.0.0.1,localhost"
}
```

`httpProxy`、`httpsProxy`、`noProxy` 分别映射到 `HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY`。
如果未单独配置 `httpsProxy`，EcoClaw 会回退使用 `httpProxy`。

### 环境变量

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `OPENROUTER_API_KEY` | 否* | — | OpenRouter API 密钥（`secrets.local.json` 未配置时使用） |
| `HTTP_PROXY` | 否 | — | PinchBench / OpenRouter 出站 HTTP 代理 |
| `HTTPS_PROXY` | 否 | `HTTP_PROXY` | 出站 HTTPS 代理 |
| `NO_PROXY` | 否 | `127.0.0.1,localhost`（启用代理时） | 绕过出站代理的主机列表 |
| `ECOCLAW_PROXY_PORT` | 否 | `8403` | 本地代理端口 |
| `ECOCLAW_EMBEDDING_MODEL` | 否 | `openai/text-embedding-3-small` | embedding 模型（构建和运行时） |

> \* 必须通过 `secrets.local.json` 或环境变量其中一种方式提供 API Key。
>
> 出站代理能力基于 Node 核心 `proxyEnv`，当前按 Node.js 24.5+ 验证。

## 🛠️ 开发

### 构建命令

```bash
npm run pull:snapshot      # 拉取 PinchBench 数据快照（需要 API Key）
npm run build              # 构建（自动先运行 build:embeddings 生成向量数据）
npm run build:embeddings   # 单独重新生成 embedding 向量（需要 API Key）
npm run dev                # watch 模式构建
```

### 测试

```bash
npm test                   # 运行测试
npm run test:watch         # watch 模式测试
npm run typecheck          # 类型检查
```

### 项目结构

```
EcoClaw/
├── src/
│   ├── index.ts                 # 插件入口、注册 provider 和 slash commands
│   ├── config.ts                # 环境变量和配置常量
│   ├── models.ts                # 模型定义（ecoclaw/best|auto|eco）
│   ├── proxy.ts                 # 本地 HTTP 代理（拦截请求、选择模型、转发 OpenRouter）
│   └── router/
│       ├── classifier.ts        # Embedding 最近邻分类器（23 个任务类别）
│       ├── embedding-engine.ts  # Embedding API 调用和余弦相似度
│       ├── embedding-data.gen.ts# 预计算的 embedding 向量（自动生成）
│       ├── selector.ts          # 基于 benchmark 分数选择最优模型
│       ├── profiles.ts          # 省钱策略配置（best/balanced/eco）
│       └── index.ts             # 模块统一导出
├── src/data/
│   ├── cache.ts                 # Benchmark 数据缓存（6 小时 TTL）
│   ├── fetcher.ts               # PinchBench API 数据拉取
│   └── snapshot.json            # 离线 benchmark 快照
├── scripts/
│   ├── build-embeddings.ts      # 从任务描述生成 embedding 向量
│   ├── pull-snapshot.ts         # 拉取 PinchBench 数据快照
│   └── print-selections.ts     # 打印各类别×策略的模型选择结果
├── openclaw.plugin.json         # OpenClaw 插件元数据
├── tsconfig.json
└── tsup.config.ts
```
